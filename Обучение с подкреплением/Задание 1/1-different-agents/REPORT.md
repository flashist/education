# Сравнение стратегий многорукого бандита: A/B тест, ε-жадная и Thompson Sampling

## 1. Задача

Задача многорукого бандита (multi-armed bandit): есть K рук (вариантов), каждая приносит случайную Бернуллиевскую награду с неизвестной вероятностью p_k. Цель — максимизировать суммарную награду (или минимизировать сожаление — regret) за T шагов.

**Regret** — суммарная упущенная выгода по сравнению с оптимальной стратегией:

```
R(T) = T * p* - Σ_{t=1}^{T} p_{a_t}
```

где `p* = max_k p_k` — вероятность лучшей руки, `a_t` — выбранная рука на шаге t.

---

## 2. Стратегии

### 2.1 A/B тест (`ABTestSampler`)

Классический двухфазный подход:

**Фаза исследования** (exploration): перебирает все руки по кругу (round-robin):
```
arm = t mod K,  при t < K * n_explore
```

**Фаза эксплуатации** (exploitation): фиксируется на лучшей руке по оценке:
```
arm = argmax_k θ̂_k,  при t ≥ K * n_explore
```

где `θ̂_k = successes_k / pulls_k` — эмпирическая оценка вероятности.

**Параметры:**
- `n_explore = 50` — количество шагов на каждую руку в фазе исследования

**Проблема:** если `n_explore` мало — оценки ненадёжны; если велико — тратится много шагов на заведомо плохие руки. Нет адаптации к накопленной информации.

---

### 2.2 ε-жадная стратегия (`EGreedy`)

На каждом шаге:
- с вероятностью ε — случайный выбор (exploration)
- с вероятностью 1−ε — жадный выбор лучшей руки (exploitation)

```
a_t = { случайный k,      с вер. ε
       { argmax_k θ̂_k,    с вер. 1−ε
```

**Параметры:**
- `epsilon = 0.1` — доля исследования
- `n_learning = 500` — количество начальных шагов чистого исследования

**Особенность:** продолжает исследовать даже после нахождения лучшей руки. Regret растёт линейно с коэффициентом ε * (p* − p̄).

---

### 2.3 Thompson Sampling (`ThompsonSampler`)

Байесовский подход. Для каждой руки поддерживает Beta-распределение:

```
θ_k ~ Beta(α_k, β_k)
```

На каждом шаге:
1. Сэмплируем `s_k ~ Beta(α_k, β_k)` для каждой руки
2. Выбираем `a_t = argmax_k s_k`
3. После получения награды r обновляем:
   - `α_k += r`
   - `β_k += 1 − r`

**Параметры:**
- `prior_alpha = 1.0`, `prior_beta = 1.0` — начальное равномерное априорное распределение (Beta(1,1))

**Преимущество:** автоматически балансирует исследование и эксплуатацию через неопределённость. По мере накопления данных posterior сужается, и выбор всё чаще падает на лучшую руку.

## 4. Описание графиков

### График 1: Mean Cumulative Reward
**Что показывает:** среднюю суммарную награду по шагам (усредненную по N_RUNS прогонам).

**Как читать:** чем выше кривая — тем лучше. Thompson Sampling набирает награду быстрее всего, так как быстро сходится к лучшей руке. A/B тест начинает медленно (тратит шаги на исследование всех рук), но затем "догоняет" за счёт фиксации на лучшей руке. ε-greedy набирает чуть меньше, так как постоянно тратит ε долю шагов на случайные выборы.

---

### График 2: Mean Cumulative Regret
**Что показывает:** среднее кумулятивное сожаление — суммарные потери по сравнению с оптимальной стратегией.

**Как читать:** чем ниже и более пологая кривая — тем лучше.

- **A/B тест:** резкий рост regret в начале (фаза исследования — тянем плохие руки), затем горизонтальное плато (нашли лучшую руку и тянем только её). Итоговый regret высокий.
- **ε-жадный:** линейный рост с постоянным наклоном (slope = ε × gap). Regret меньше, чем у A/B теста при правильном ε.
- **Thompson Sampling:** сначала небольшой рост (нужно чуть разведать), потом почти горизонтально. Наименьший итоговый regret.

---

### График 3–5: Actions Scatter (по одному для каждой стратегии)
**Что показывает:** какую руку выбирал агент на каждом шаге одного прогона (seed=42). Цветные точки — руки, ось X — шаг, ось Y — номер руки.

**Как читать:**

- **A/B тест:** первые `K * n_explore = 500` шагов — равномерное чередование всех рук (горизонтальные полосы). После шага 500 — все точки на одной строке (лучшая рука).
- **ε-жадный:** большинство точек на лучшей руке, но периодически появляются точки на других руках (случайные исследования с вероятностью ε).
- **Thompson Sampling:** вначале активное исследование многих рук, постепенное сужение к одной (лучшей). Более "умное" исследование: чаще трогает перспективные руки.

**Что показывает:** гистограмму распределения итоговой суммарной награды по всем N_RUNS прогонам для каждой стратегии.

**Как читать:** чем правее пик гистограммы — тем выше средняя награда. Thompson Sampling имеет пик правее остальных и меньший разброс (более стабилен). A/B тест может иметь широкий разброс при небольшом n_explore.

## 5. Вывод

Thompson Sampling превосходит A/B тест и ε-жадную стратегию по следующим причинам:

1. **Адаптивное исследование:** Thompson Sampling автоматически исследует руки пропорционально их неопределённости. Плохие руки исследуются редко, перспективные — чаще.

2. **Нет фиксированного бюджета исследования:** A/B тест тратит ровно `K * n_explore` шагов независимо от результатов. Thompson Sampling прекращает исследовать руку, как только убеждается в её неоптимальности.

3. **Нет постоянных потерь:** ε-жадная стратегия всегда тратит долю ε на случайный выбор, даже если лучшая рука известна наверняка. Thompson Sampling стремится к 0 по мере накопления данных.

4. **Байесовская когерентность:** posterior Beta-распределение корректно учитывает всю накопленную информацию, а не только точечную оценку.

**Практический итог:** при тех же N_TRIALS шагах Thompson Sampling в среднем получает больше наград и оставляет наименьший regret.

---

### Наблюдение: при хорошо подобранном n_explore A/B тест может конкурировать с Thompson Sampling

В данном эксперименте на гистограмме суммарных наград A/B тест показывает результат, сопоставимый с Thompson Sampling. Это объясняется совокупностью факторов:

- **Малая доля исследования.** При `n_explore = 50` и `K = 10` A/B тест тратит `500` шагов на исследование — всего **5% от N_TRIALS = 10 000**. Оставшиеся 9 500 шагов агент жёстко фиксируется на лучшей руке.

- **Хорошая различимость лучшей руки.** Лучшая рука (p = 0.11) почти вдвое превосходит вторую по качеству (p = 0.06). 50 испытаний на каждую руку достаточно, чтобы надёжно идентифицировать победителя, и A/B тест корректно переключается на неё.

- **Медленная сходимость Thompson при малых payout'ах.** Все вероятности успеха меньше 0.12 — награды редкие. Beta-апостериоры для разных рук медленно расходятся, и Thompson Sampling продолжает изредка исследовать субоптимальные руки на протяжении всего горизонта. В результате за 10 000 шагов Thompson реально проводит на лучшей руке ~85–90% времени, а не 95%, как A/B тест после фазы исследования.

**Вывод:** превосходство Thompson Sampling над A/B тестом проявляется асимптотически (при большом T) или когда `n_explore` плохо подобран. При конечном горизонте с удачно выбранным `n_explore` и хорошо различимым оптимумом A/B тест вполне конкурентен — что и наблюдается в данном эксперименте. Thompson Sampling выигрывает на кривой regret (более пологий рост), но суммарная награда за T = 10 000 шагов оказывается близкой.
